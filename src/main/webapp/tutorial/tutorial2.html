Histórias


A elaboração de histórias de usuário é uma das atividades fundamentais no XP. As histórias de usuário descrevem cenários com situações de utilização que os envolvidos gostariam que o sistema em desenvolvimento viesse a oferecer. Elas devem ser escritas preferencialmente pelos próprios usuários. As histórias de usuário são semelhantes aos casos de uso da UML, mas não são a mesma coisa. A diferença é que elas não se limitam a descrever o processo de interação do usuário com o sistema. As histórias não têm a pretensão de armazenar todas as informações. No desenvolvimento ágil é dado mais valor ao dialogo presencial direto com o cliente par aprender o maximo possivel. As histórias no sistema visa ter o minimo necessário para poder colocar em pratica a mesma.

Nessa tela é para ser criadas e priorizadas histórias pelo cliente, com as funcionalidades que ele deseja no sistema , para o próximo Sprint (Iteração). Ele deve fazer isso com a ajuda da equipe que vai desenvolver. Logo após a criação e as estimativas, a equipe deve estimar as historias em pontos ou horas.
